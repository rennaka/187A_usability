1. Visibility of System Status

According to the Nielsen's 10 criteria, "the system should always keep users informed about what is going on, through appropriate feedback within reasonable time".

When you go to the library page of the university, the title and the pathway will be missing. You don't know exactly where you are.

According to Nielsen's criteria, "the system should always keep users informed about what is going on". So, on this specific page this criteria is not respected.

In addition, we tried to find the hours of operation, but before we found it, you have to read all main sections to see it. The information architecture is not well organized and so the main information is hidden.

This menu bar is the small one at the top of the page, just above the main red navigation bar. We think that the location is a bad move (lack of visibility).

By mousing over the different labels on the top navigation bar, we don't know which one is activated. None of these links are being colored by mousing over. As a result, we have to click in order to see if we are in the right page.

The change in styling when hovering over a link in the main navigation bar provides visibility of the navigation bar status. We are visually aware of which link we will click on, rather than having to guess. Unfortunately, the link blends back in with the others once the action is completed. Keeping the hover styling once we get to that page would be a nice, visual way of indicating where we are.

Instead, when we enter the site by the navigation bar on the home page, the hover styling from the home navigation bar becomes the default styling. This is a clever shift that visually indicates that we are now in the site, but once we are there, the navigation bar does not give any other useful visual cues besides the hover effect.
