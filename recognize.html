6. Recognition Rather Than Recall

"Minimize the user's memory load by making objects, actions, and options visible. The user should not have to remember information from one part of the dialogue to another. Instructions for use of the system should be visible or easily retrievable whenever appropriate".

What's the difference between recognition and recall? For instance, recognition consists of remembering have already seen this person whereas recall consists of remembering her name.

So, what's why recognition is better than recall for website designers: the users don't have to remember their actions. The pages on this site have labels that indicate where you are but you don't remember how you got there (example university honors program page). The user is forced to remember how he got there himself. And we know that recognition criteria is better than recall criteria. Human can only remember 7+-2 items in the same time.

Create a pathway at the top of each page could be a good solution to improve the recognition criteria.
